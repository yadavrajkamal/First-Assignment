Full Stack Web Development Tech Stack

Frontend (Client-Side):
HTML (Hypertext Markup Language):
The standard markup language used to create the structure of web pages.
CSS (Cascading Style Sheets):
Styles and layouts the web pages, making them visually appealing.
JavaScript:
A scripting language that adds interactivity to web pages. Can manipulate HTML and CSS in real-time.
Frontend Frameworks/Libraries:
React.js:
A JavaScript library for building user interfaces, especially single-page applications.
Vue.js:
A progressive JavaScript framework for building user interfaces and single-page applications.
Angular:
A TypeScript-based open-source web application framework led by the Angular Team at Google.
Backend (Server-Side):
Node.js:
A JavaScript runtime built on Chrome's V8 engine that allows executing JavaScript server-side.
Express.js:
A minimalist web framework for Node.js, used to build web applications and APIs.
Django:
A high-level Python web framework that encourages rapid development and clean, pragmatic design.
Ruby on Rails:
A server-side web application framework written in Ruby, designed to simplify and speed up web application development.
Databases:
MySQL:
An open-source relational database management system that uses SQL (Structured Query Language).
MongoDB:
A NoSQL database that stores data in JSON-like documents, making it flexible and scalable.
PostgreSQL:
An open-source, powerful, and feature-rich relational database management system.
Version Control:
Git:
A distributed version control system for tracking changes in source code during software development.
GitHub/GitLab/Bitbucket:
Web-based platforms for hosting Git repositories and collaborating on code.
Web Servers:
Apache:
A popular open-source HTTP server that allows hosting of websites.
Nginx:
A high-performance HTTP server and reverse proxy, as well as an IMAP/POP3 proxy server.
DevOps/Deployment:
Docker:
A platform that uses OS-level virtualization to deliver software in packages called containers.
Kubernetes:
An open-source platform designed to automate deploying, scaling, and operating application containers.
Jenkins:
An open-source automation server used to automate parts of software development related to building, testing, and deploying.
APIs (Application Programming Interfaces):
REST (Representational State Transfer):
A set of principles for designing networked applications, using stateless protocols and standard operations like GET, POST, PUT, DELETE.
GraphQL:
A query language for APIs that allows clients to request only the data they need.
Authentication & Authorization:
OAuth:
An open standard for token-based authentication and authorization on the internet.
JWT (JSON Web Token):
A compact, URL-safe means of representing claims to be transferred between two parties.
Testing:
Jest:
A JavaScript testing framework maintained by Facebook, often used with React.
Mocha:
A JavaScript test framework running on Node.js, featuring browser support, asynchronous testing, and more.
Selenium:
An open-source tool for automating web browsers, often used for testing web applications.
Package Managers:
npm (Node Package Manager):
A package manager for JavaScript, included with Node.js, used for managing dependencies.
Yarn:
An alternative package manager for JavaScript that focuses on speed, security, and consistency.
Build Tools:
Webpack:
A module bundler for JavaScript applications, used to bundle and serve web assets.
Babel:
A JavaScript compiler that allows using next-generation JavaScript syntax.
